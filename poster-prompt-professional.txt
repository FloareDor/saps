① The Problem

Text: Diffusion LLMs denoise text over T steps, holding the full KV cache in memory at every step. Sparse-dLLM prunes it, but keeps the same fraction of tokens at every step. That is the key limitation.

Diagram: Clean pen-style illustration of repeated token rows of identical size across multiple denoising steps, labeled: "same budget every step"

---

② The Insight: Context Needs Change Over Time

Text: Early steps form global structure such as topic and sentence-level relationships, so they need broader context. Late steps mostly refine local details, so much distal context becomes less useful.

Bullet points:
- Early steps: keep 70%
- Late steps: keep 10%

Diagram: Two clean bar outlines — a tall bar labeled Early, 70% and a short bar labeled Late, 10%

---

③ SAPS: The Schedule

Text: SAPS replaces the fixed pruning ratio with a smooth exponential decay over denoising steps. It requires no model retraining, only a patch to the Sparse-dLLM inference pipeline, with essentially negligible overhead.

Equation: r(t) = r_max · (r_min / r_max)^(t / (T - 1)), r_max = 0.7, r_min = 0.1

Diagram: A clean pen-style graph comparing a flat dashed line for Sparse-dLLM and a smooth decaying curve for SAPS. Annotate the curve with 0.7 at the start and 0.1 at the end. Add a small neat annotation: "Overhead: one floating-point op per step"

④ Results (LLaDA-8B, GSM8K, n = 1319)

Method        | KV Cache | Accuracy
--------------|----------|----------
Vanilla LLaDA | 145 MB   | 78.17%
Sparse-dLLM   | 72 MB    | 76.3%
SAPS (ours)   | 50 MB    | 78.2%

Text: SAPS matches vanilla accuracy while using 66% less memory than vanilla and 31.4% less memory than Sparse-dLLM. It is a Pareto improvement: lower memory and higher accuracy.

⑤ Why It Works

Text: Keeping more context early helps the model form stronger global structure for multi-step reasoning. Pruning aggressively late focuses attention on the most salient tokens, which acts like inference-time regularization.

Additional note: Code benchmarks such as HumanEval and MBPP show SAPS and Sparse are roughly tied. The main gains appear in math reasoning.

---

⑥ Ablations & Future Work

Bullet points:
- Layer-aware schedule r(t, l): degrades accuracy on both math and code
- Softer r_min = 0.2: hurts code benchmarks, so the harder floor matters

Future work:
